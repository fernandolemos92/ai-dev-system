# SKILL: define-growth-strategy

## Purpose

Perform the explicit bounded synthesis step for the `growth-strategy` workflow.

This skill exists to:

- enter the governed growth-strategy workflow through a real bounded traversal step
- identify which growth specialists are materially relevant for the active execution line
- resolve and use those specialists honestly through the registry/runtime bridge
- synthesize bounded strategic direction that can guide downstream work
- determine whether a durable `Strategy` artifact is required
- close the bounded growth-strategy moment cleanly and stop

This skill is the canonical first-pass synthesis step for governed growth-strategy work.

It is not a generic brainstormer.

It is not a substitute for governance, live state, workflow authority, or implementation.

It is not full go-to-market planning.

---

## When To Use

Use this skill when all of the following are materially true:

- the active execution line is inside a governed path where `growth-strategy` has been explicitly evaluated as materially required
- the system needs strategic communication, discoverability, funnel, or measurement direction before safe downstream progression
- the current need is to synthesize growth-facing direction rather than execute copy, SEO, CRO, analytics, or implementation work
- the bounded step still belongs to strategy direction, not execution

Typical triggers include:

- `discover-missing-context` or `structured-research` concluded that growth-strategy is materially required
- `product-definition` or another governing workflow needs explicit strategic synthesis before safe PRD or later progression
- the product includes messaging-sensitive, acquisition-sensitive, discoverability-sensitive, or measurement-sensitive surfaces
- downstream tasking, handoff, review, or implementation would otherwise rely on fuzzy value framing or ungrounded strategic assumptions

---

## When Not To Use

Do not use this skill when:

- the current need is still raw contextual discovery
- structured research is still materially required and has not yet completed
- the current need is execution of copy, SEO, CRO, analytics, or channel work
- the scope is purely backend or non-market-facing
- the strategic framing is already sufficiently governed for the same execution line and no new bounded synthesis is needed
- the need is product positioning exploration without a governed strategic decision context

This skill does not replace:

- `discover-missing-context`
- `define-prd`
- `plan-tasks`
- `prepare-handoff`
- copy execution
- SEO execution
- CRO execution
- analytics implementation

---

## Governing Workflow Relationship

This skill executes inside the governing workflow:

`system/workflows/growth-strategy.md`

It is the canonical bounded traversal entry for first-pass governed growth-strategy synthesis.

Accordingly, this skill must:

- treat the workflow as real authority, not decorative context
- remain subordinate to governance, project state, active artifacts, and the controlling workflow
- honor the workflow requirement that strategic direction be materially synthesized rather than merely acknowledged
- stop after the bounded growth-strategy moment closes

This skill must not behave as though specialist naming alone satisfies workflow traversal.

It must not treat scattered “growth ideas” as equivalent to bounded strategic synthesis.

---

## Core Responsibility

This skill is responsible for answering, in bounded synthesized form:

- what strategic question needs direction now
- which growth specialists are materially relevant
- what positioning, messaging, discoverability, funnel, or measurement direction should guide downstream work
- what remains grounded versus still inferential
- whether a durable `Strategy` artifact is required
- what the correct next governed move becomes after this growth-strategy moment closes

This skill must produce explicit synthesis.

It must not stop at:

- vague growth language
- hype-heavy positioning
- unsynthesized specialist fragments
- loose idea lists
- execution planning

---

## Research and Definition Boundary

This skill assumes that foundational context is already sufficiently clear for a bounded growth-strategy pass.

When used before PRD safety is fully established, it still assumes that structured research already materially completed if research was required upstream.

It may use existing context, research, PRD, validation, and active workflow signals.

It must not reopen broad contextual discovery unless a blocking inconsistency prevents safe synthesis.

If a blocking inconsistency exists, this skill must stop honestly and return control rather than compensating by guessing.

---

## Mandatory Specialist Evaluation

This skill must explicitly evaluate which of the following specialists are materially relevant for the current bounded pass:

- `positioning`
- `copy-messaging`
- `seo`
- `cro`
- `analytics`

This skill must not pretend all specialists were equally needed.

It should identify:

- which specialists were materially required
- which specialists were actually used
- which specialists were non-blocking for this bounded pass
- whether any required specialist remained unavailable or only partially successful

Silent omission is not valid.

For messaging-sensitive or acquisition-sensitive surfaces, `positioning` should normally be explicitly evaluated.

When value framing is materially unclear, `positioning` should normally be treated as materially relevant unless a sufficiently stable governed equivalent already exists for the same execution line.

In those cases, this skill must be able to state explicitly one of the following:

- `positioning` was materially required and used
- `positioning` was explicitly evaluated and a governed equivalent already exists
- `positioning` was evaluated and found genuinely non-blocking for this bounded pass

---

## Specialist Invocation Discipline

When a specialist is materially required, this skill must preserve the canonical registry/runtime bridge.

That means:

- resolve the canonical project agent first
- use the registry-approved runtime type and runtime call target
- do not send the canonical agent name directly as the runtime call target when registry mapping says otherwise
- report specialist participation honestly as:
  - materially succeeded
  - partial / behavioral
  - failed
  - unavailable
  - not materially needed for this bounded pass

This skill must not narrate specialist participation as materially complete when routing or execution failed.

It must not collapse:

- routed specialist identity
- runtime target
- attempted specialist use
- successful specialist contribution

into one overclaimed narrative.

Successful routed or adapted specialist use inside this bounded step does not, by itself, satisfy later strategic execution work.

---

## Inputs This Skill Should Use

When available and materially relevant, this skill should ground its synthesis in:

- `project/PROJECT_STATE.md`
- the currently governing workflow context
- stabilized contextual understanding
- research findings
- active PRD
- active validation
- an active `Strategy` artifact if one already exists for the same execution line
- known buyer and user profile when already clarified
- known competitive framing
- known trust, acquisition, or discoverability context
- repository continuity for the same product line when clearly governing the same execution line

This skill must not invent missing upstream truth.

If a supposedly governing input is absent, contradictory, or insufficient for safe synthesis, this skill must stop honestly rather than fabricate strategic direction.

---

## Minimum Useful Strategic Direction

A minimally useful contribution from this skill should normally leave behind at least a bounded strategic foundation such as:

- product frame direction
- audience-fit direction
- value-emphasis direction
- grounded versus weak differentiation signals
- communication constraints or truthfulness boundaries
- discoverability, funnel, or analytics stance when materially relevant

A contribution that improves tone or ambition but does not materially improve downstream strategic clarity should be reported conservatively.

---

## What This Skill Should Synthesize

A strong bounded pass from this skill should usually synthesize some combination of:

### 1. Strategic frame
- what the product is in market-facing terms
- what category or mental model best reduces confusion
- what angle deserves emphasis
- what should remain secondary

### 2. Audience fit
- who this is most clearly for
- whose understanding matters most right now
- what context makes the value legible
- where buyer and end-user priorities align or diverge when relevant

### 3. Messaging direction
- what must be communicated clearly
- what should remain consistent downstream
- what language should stay conservative
- what message direction is grounded versus still open

### 4. Discoverability / funnel / measurement direction
- whether SEO materially matters now
- whether CRO materially matters now
- whether analytics materially matters now
- what downstream assumptions must be visible rather than improvised

### 5. Strategic boundaries
- what must not be overclaimed
- what remains assumption versus grounded direction
- what should stay open pending stronger evidence

This skill must remain direction-level.

It must not drift into campaign strategy, channel planning, SEO implementation, analytics instrumentation, copy execution, or code.

---

## Single-Frame Preference Rule

When this skill is used before durable `Strategy` closure or before later downstream work depends on strategic framing, it should normally prefer one bounded primary strategic frame over multiple equally weighted alternatives unless the governed step explicitly asks for option exploration.

This rule exists to reduce ambiguity for downstream work.

If multiple viable frames exist, this skill should still identify one primary recommended frame and mark the rest as secondary or still-open rather than leaving all options equally unresolved.

---

## Truthfulness Rule

This skill must clearly separate:

- directly grounded strategic direction
- plausible strategic inference
- still-open strategic uncertainty

It must not disguise aspirational or convenient framing as validated truth.

A plausible strategic direction may still be useful.

But it must remain honestly framed as direction, not proof.

This matters especially for:

- positioning statements
- message priority
- CTA logic
- SEO or discoverability implications
- conversion assumptions
- analytics assumptions
- trust or authority claims

This skill must not silently harden fabricated precision such as:

- conversion rates
- ranking expectations
- market-size assumptions
- audience-size assumptions
- exact ICP segmentation
- exact messaging-performance claims
- exact funnel percentages

unless those points are already materially grounded upstream.

---

## Artifact Decision Rule

This skill must explicitly determine whether a durable `Strategy` artifact is required.

A durable `Strategy` artifact is usually required when one or more of the following is true:

- multiple later steps will depend on the same strategic direction
- conversational strategic guidance would be too fragile for the current scope
- downstream tasking, handoff, implementation, or review quality would materially benefit from durable strategic assumptions
- the execution line includes meaningful acquisition, discoverability, conversion, or measurement surfaces
- stable communication or positioning direction needs to survive across later stages

If a durable `Strategy` artifact is required, this skill must produce output compatible with canonical materialization in:

`project/strategy/active/`

This skill must not treat a durable artifact as complete if it was only implied conversationally and not materially created when required.

If an artifact is not required for the current bounded pass, this skill must say so explicitly.

---

## Single Active Strategy Artifact Rule

When a durable `Strategy` artifact is required for the current execution line, this skill must preserve a single active artifact truth for the `strategy` family.

This skill must not leave multiple active `Strategy` artifacts in `project/strategy/active/` for the same execution line by default.

If an older active `Strategy` artifact is being superseded by a newer bounded synthesis for the same execution line, this skill must do one of the following coherently:

- update the existing active artifact in place
- replace the active artifact and retire the older one from `active/`
- stop and report lifecycle closure as incomplete if supersession cannot be handled cleanly

This skill must not treat “new artifact created” as valid closure when the result leaves parallel active `Strategy` artifacts that claim to govern the same execution line.

---

## Output Modes

This skill has only two valid non-error output modes.

### Output Mode A — Growth Strategy Continues / Still Blocked

Use this mode when:

- a blocking prerequisite is still missing
- a materially required specialist could not be safely used
- the bounded pass cannot yet produce honest synthesis
- durable closure would be fake or incomplete

A correct output in this mode must include:

1. a short assessment of what blocks safe growth-strategy synthesis
2. which specialist or prerequisite is blocking
3. what remains decided vs unresolved
4. the correct next bounded move
5. a clear stop

This mode must not compensate for the blockage with vague marketing advice.

---

### Output Mode B — Growth Strategy Defined For Current Pass

Use this mode when bounded synthesis materially succeeded for the current pass.

A correct output in this mode must include:

1. a concise statement that growth-strategy materially completed for the current bounded pass
2. which specialists were materially relevant
3. which specialists were actually used and with what execution outcome
4. the synthesized direction
5. whether a durable `Strategy` artifact is required and whether it was created
6. the correct next allowed governed move
7. a clear stop without executing that next move inline

---

## Canonical Output Shape

When this skill materially succeeds, the output should normally make explicit:

- Governing Workflow: growth-strategy
- Bounded Step: define-growth-strategy
- Strategic Focus: [what needed direction]
- Material Specialists Required: [list]
- Specialists Actually Used: [list + honest outcome]
- Product Frame Direction: [bounded synthesis]
- Messaging / Discoverability / Funnel Direction: [bounded synthesis]
- Strategic Truthfulness Notes: [grounded vs inferred vs still open]
- Strategy Artifact: [required and created | not required for this pass | blocked]
- Downstream Consequence: [what later work must now reflect]
- Next Allowed Step: [single bounded governed move]
- Stop: yes

For messaging-sensitive or acquisition-sensitive surfaces, the output should not omit explicit treatment of `positioning` evaluation.

---

## Anti-Premature Progression

This skill must actively prevent growth-strategy from collapsing into later-stage work.

It must not:

- auto-open `experience-direction`
- auto-open `plan-tasks`
- auto-open `prepare-handoff`
- auto-open implementation
- auto-open copy execution, SEO execution, CRO execution, or analytics implementation
- treat improved strategic clarity as permission for same-turn downstream continuation
- generate code
- choose implementation baselines on its own
- invent campaign plans, channel plans, or GTM theater as though the bounded step required them
- substitute hype or option sprawl for synthesized direction
- treat specialist consideration as equivalent to specialist success

If this bounded step materially completes, it must stop after:

- producing explicit synthesis
- creating the durable `Strategy` artifact when required
- applying coherent state closure when valid
- identifying the next allowed governed move

Nothing later.

---

## State Closure Expectations

If this skill materially completes and coherent state closure is possible, the resulting `PROJECT_STATE.md` should describe one coherent post-strategy moment.

At minimum, that state should align with:

- `Current Phase = strategy`
- the real `Active Strategy` artifact when one exists
- whether growth-strategy materially completed for the current pass
- the correct next governed move
- the fact that same-turn continuation is not authorized

The resulting state must not point to an `Active Strategy` artifact while another competing active artifact remains in `project/strategy/active/` for the same execution line.

If artifact supersession is unresolved, closure remains operationally incomplete.

If coherent state closure is not possible, this skill must report the closure as operationally incomplete rather than pretending the transition fully passed.

---

## Boundaries

This skill must not:

- behave like `/start`
- reopen broad discovery by default
- create a PRD
- create validation scenarios
- plan tasks
- prepare a handoff
- authorize implementation
- implement anything
- replace governance, live state, or workflow authority
- replace copy execution, SEO execution, CRO execution, or analytics implementation

Its role is limited to:

- bounded strategic synthesis
- honest specialist evaluation and use
- durable direction decision for the current pass
- clean closure of the bounded growth-strategy moment

---

## Failure Handling

If a materially required specialist cannot be safely routed or executed:

- do not narrate that specialist as successfully used
- report the limitation clearly
- downgrade the synthesis claim conservatively
- stop if the workflow remains materially blocked

If inputs are too weak or contradictory for safe synthesis:

- do not compensate by inventing direction
- report the blocking ambiguity
- identify the correct prior or adjacent governed step
- stop

If a durable `Strategy` artifact is materially required but cannot be created canonically:

- do not narrate growth-strategy as fully closed
- report lifecycle closure as incomplete
- stop

If strategic truthfulness remains too weak because core claims are still mostly speculative:

- do not convert aspiration into settled direction
- report what remains open
- stop or downgrade closure conservatively as needed

---

## Final Rule

A correct `define-growth-strategy` must answer, in bounded synthesized form:

- what strategic question needed direction?
- which growth specialists were materially relevant?
- what usable strategic direction now exists?
- whether a durable `Strategy` artifact is required?
- what is the single next allowed governed move?

It must then stop.

It must not turn growth-strategy into marketing theater, execution planning, or hype-heavy improvisation without durable downstream value.
